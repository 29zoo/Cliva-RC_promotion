# 실행 방법 (터미널)

프로젝트 루트: `d:\Cliva RC\Promotion`  
DB·서버 관례: **KOIHA-NTRH cert-quiz-package** 와 동일 (PostgreSQL)

---

## 0. 사전 확인

```powershell
node -v    # v22+
npm -v
```

PostgreSQL 로컬 개발 시 DB 생성:

```powershell
# psql 접속 후
# CREATE DATABASE booth_roulette;
```

---

## 1. 최초 설정 (1회)

```powershell
cd "d:\Cliva RC\Promotion\booth-roulette"

npm install
copy server\.env.example server\.env
copy client\.env.example client\.env
```

`server\.env` 【입력】 예:

```env
DATABASE_URL="postgresql://postgres:비밀번호@localhost:5432/booth_roulette?schema=public"
NODE_ENV=development
PORT=4001
```

```powershell
npm run db:migrate
npm run db:seed
```

`client\.env` 【입력】: `VITE_PROMO_VIDEO_URL` (선택)

---

## 2. 개발 실행

```powershell
cd "d:\Cliva RC\Promotion\booth-roulette"
npm run dev
```

| 용도 | URL |
|------|-----|
| 부스 화면 | http://localhost:5180 |
| API | http://localhost:4001 (`server\.env` PORT) |
| 헬스체크 | http://localhost:4001/api/health |

종료: `Ctrl+C`

---

## 3. 운영 실행 (로컬 PC)

```powershell
cd "d:\Cliva RC\Promotion\booth-roulette"
npm run build
cd server
npm start
```

`server\.env` 운영 예:

```env
NODE_ENV=production
CORS_ORIGIN=https://promotion.cliva.net
TRUST_PROXY=true
PORT=4001
```

접속: http://localhost:4001

## 8. 문제 해결

**API 응답**

```powershell
Invoke-RestMethod http://127.0.0.1:4001/api/health
```

**PostgreSQL 연결**

```powershell
# server\.env DATABASE_URL 사용자·비밀번호·DB명 확인
npm run db:migrate
```

**운영 기동 실패 (CORS)**

```powershell
# server\.env 에 NODE_ENV=production 이면 CORS_ORIGIN 【입력】 필수
```

**포트 확인**

```powershell
netstat -ano | findstr :4001
netstat -ano | findstr :5180
```
