## 1. 최초 설정 (1회)

```powershell
cd "d:\Cliva RC\Promotion\booth-roulette"

npm install
copy server\.env.example server\.env
copy client\.env.example client\.env
```

`server\.env` 【입력】 예 (로컬 Docker, 포트 5433):

```env
DATABASE_URL="postgresql://postgres:postgres@localhost:5433/booth_roulette?schema=public"
NODE_ENV=development
CORS_ORIGIN=
TRUST_PROXY=false
HOST=0.0.0.0
PORT=4001
LOG_LEVEL=info
```

```powershell
npm run db:migrate
npm run db:seed
```

`client\.env` 【입력】:

```env
# Nginx /api 프록시 없이 dev만 쓸 때도 보통 비움 (Vite가 /api·/uploads 프록시)
VITE_API_BASE_URL=
```

> **홍보 영상**은 env가 아니라 **관리자 화면 → 🎬 홍보 영상 → MP4 업로드**로 등록합니다.

## 8. 문제 해결

**API 응답**

```powershell
Invoke-RestMethod http://127.0.0.1:4001/api/health
Invoke-RestMethod http://127.0.0.1:4001/api/booth/state
```

**PostgreSQL 연결 (`postgresql://` 프로토콜 오류)**

- `server\.env` 의 `DATABASE_URL`이 `file:./dev.db` (SQLite) 형식이면 **PostgreSQL URL**로 바꿉니다.
- Docker Postgres 꺼짐 → Docker Desktop 실행 후 `docker ps` 로 컨테이너 확인.

```powershell
npm run db:migrate
```

**홍보 영상이 재생되지 않음**

- 관리자 화면에서 영상 업로드 여부 확인
- dev: Vite가 `/uploads` 를 API(4001)로 프록시 — `npm run dev` 재시작
- 운영: Nginx에 `/uploads/` 프록시 필요 ([서버업로드 관리.md](서버업로드%20관리.md) 참고)

**운영 기동 실패 (CORS)**

```powershell
# server\.env 에 NODE_ENV=production 이면 CORS_ORIGIN 【입력】 필수
```

**포트 확인**

```powershell
netstat -ano | findstr :4001
netstat -ano | findstr :5180
```
