# 서버 업로드 · 관리

원격 Linux 서버에 `booth-roulette`를 올려 운영할 때 사용합니다.  
로컬 PC만 쓰는 경우 → [실행방법.md](실행방법.md)

---

## 1. 서버 요구사항

| 항목 | 내용 |
|------|------|
| OS | Ubuntu 22.04+ 등 Linux |
| Node.js | v22 LTS |
| 포트 | 4000 (또는 `.env`의 `PORT`) |
| DB | SQLite (`server/prisma/dev.db`) |

Node 설치 (서버):

```bash
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs
node -v && npm -v
```

---

## 2. 업로드 대상

서버에 올릴 폴더: **`booth-roulette/`**

제외 (업로드·압축 시 빼기):

```
node_modules/
server/.env          # 서버에서 별도 생성
server/prisma/dev.db # 운영 DB는 서버에서 생성
client/dist/         # 서버에서 build 하면 생략 가능
.git/
```

## 3. 로컬 → 서버 업로드

### 3-1. SCP (Windows PowerShell)

```powershell
cd "d:\Cliva RC\Promotion"

# booth-roulette 전체 (node_modules 제외 권장)
scp -r booth-roulette user@서버IP:/home/user/
```

`node_modules` 없이 보내려면 서버에서 `npm install` 실행.

### 3-2. rsync (WSL · Git Bash)

```bash
rsync -avz --exclude node_modules --exclude server/.env --exclude server/prisma/dev.db \
  "/d/Cliva RC/Promotion/booth-roulette/" \
  user@서버IP:/home/user/booth-roulette/
```

### 3-3. 압축 후 업로드

```powershell
cd "d:\Cliva RC\Promotion"
tar -czf booth-roulette.tar.gz --exclude=node_modules --exclude=server\.env booth-roulette
scp booth-roulette.tar.gz user@서버IP:/home/user/
```

서버에서:

```bash
cd /home/user
tar -xzf booth-roulette.tar.gz
```

## 4. 서버 최초 배포

```bash
cd /home/user/booth-roulette

npm install

cp server/.env.example server/.env
cp client/.env.example client/.env
# server/.env · client/.env 의 【입력】 항목 작성 (각 .env.example 주석 참고)

npm run db:migrate
npm run build
```

환경 변수 상세: `server/.env.example`, `client/.env.example`

동작 확인:

```bash
cd server
npm start
# 다른 터미널
curl http://127.0.0.1:4000/api/health
```

브라우저: `http://서버IP:4000`

## 5. PM2로 상시 실행

```bash
sudo npm install -g pm2

cd /home/user/booth-roulette/server
pm2 start dist/src/index.js --name booth-roulette
pm2 save
pm2 startup    # 부팅 시 자동 시작 (출력된 명령 실행)
```

| 명령 | 설명 |
|------|------|
| `pm2 status` | 상태 |
| `pm2 logs booth-roulette` | 로그 |
| `pm2 restart booth-roulette` | 재시작 |
| `pm2 stop booth-roulette` | 중지 |
| `pm2 delete booth-roulette` | 제거 |

## 6. 업데이트 (재배포)


## 7. DB 백업 · 복원

DB 파일: `server/prisma/dev.db`

**백업**

```bash
cd /home/user/booth-roulette/server/prisma
cp dev.db "dev.db.bak.$(date +%Y%m%d_%H%M%S)"
```

로컬로 내려받기:

```powershell
scp user@서버IP:/home/user/booth-roulette/server/prisma/dev.db "d:\Cliva RC\Promotion\backup\dev.db"
```

**복원**

```bash
pm2 stop booth-roulette
cd /home/user/booth-roulette/server/prisma
cp dev.db.bak.YYYYMMDD_HHMMSS dev.db
pm2 start booth-roulette
```

## 8. 방화벽 · 포트

```bash
sudo ufw allow 4000/tcp
sudo ufw status
```

Nginx 리버스 프록시 (선택, 80 → 4000):

```bash
sudo apt install nginx -y
sudo nano /etc/nginx/sites-available/booth-roulette
```

```nginx
server {
    listen 80;
    server_name your-domain.or.ip;

    location / {
        proxy_pass http://127.0.0.1:4000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

```bash
sudo ln -s /etc/nginx/sites-available/booth-roulette /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
```

## 10. 트러블슈팅

**502 / 연결 안 됨**

```bash
pm2 status
ss -tlnp | grep 4000
pm2 logs booth-roulette --err
```

**DB locked**

```bash
pm2 stop booth-roulette
cd /home/user/booth-roulette/server/prisma
rm -f dev.db-journal
cd ../..
npm run db:migrate
pm2 start booth-roulette
```

**빌드 후 화면 안 뜸**

```bash
ls -la /home/user/booth-roulette/client/dist/index.html
npm run build
pm2 restart booth-roulette
```
