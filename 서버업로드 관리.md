## 4. env 설정

```bash
cd /var/www/Cliva-RC_promotion/booth-roulette

cp server/.env.example server/.env
cp client/.env.example client/.env
nano server/.env
nano client/.env
```

`server/.env` 【입력】 (KOIHA `server/.env` 의 DB 계정·비밀번호 참고):

```env
DATABASE_URL="postgresql://USER:PASSWORD@localhost:5432/booth_roulette?schema=public"
NODE_ENV=production
CORS_ORIGIN=https://promotion.cliva.net
TRUST_PROXY=true
HOST=0.0.0.0
PORT=4001
LOG_LEVEL=info
```

`client/.env` 【입력】:

```env
VITE_PROMO_VIDEO_URL=https://www.youtube.com/embed/영상ID
VITE_API_BASE_URL=
```

동일 Nginx 도메인 + `/api` 프록시면 `VITE_API_BASE_URL` 비움.

## 7. Nginx (KOIHA nginx-cliva-edu.example.conf 참고)

부스 전용 서브도메인 예 (`promotion.cliva.net` → `127.0.0.1:4001`):

```nginx
upstream booth_api {
    server 127.0.0.1:4001;
    keepalive 16;
}

server {
    listen 443 ssl http2;
    server_name promotion.cliva.net;

    root /var/www/Cliva-RC_promotion/booth-roulette/client/dist;
    index index.html;

    location /api/ {
        proxy_http_version 1.1;
        proxy_set_header Host              $host;
        proxy_set_header X-Real-IP         $remote_addr;
        proxy_set_header X-Forwarded-For   $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_pass http://booth_api;
    }

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

```bash
sudo nginx -t && sudo systemctl reload nginx
curl -sS https://promotion.cliva.net/api/health
```
