## 4. env 설정

```bash
cd /var/www/Cliva-RC_promotion/booth-roulette

cp server/.env.example server/.env
cp client/.env.example client/.env
nano server/.env
nano client/.env
```

`server/.env` 【입력】 (KOIHA `server/.env` 의 DB 계정·비밀번호 참고):

```env
DATABASE_URL="postgresql://USER:PASSWORD@localhost:5432/booth_roulette?schema=public"
NODE_ENV=production
CORS_ORIGIN=https://promotion.cliva.net
TRUST_PROXY=true
HOST=0.0.0.0
PORT=4001
LOG_LEVEL=info
```

`client/.env` 【입력】:

```env
# 동일 Nginx 도메인 + /api·/uploads 프록시 → 비움
VITE_API_BASE_URL=
```

> **홍보 영상**은 env가 아닙니다. 배포 후 **관리자 화면 → 🎬 홍보 영상**에서 MP4를 업로드하세요.  
> 파일은 `server/uploads/promo/` 에 저장되며, 재배포(`git pull`) 시 **유지**됩니다 (git 제외).

## 5. 빌드 · 마이그레이션

```bash
cd /var/www/Cliva-RC_promotion/booth-roulette

npm ci
npm run db:migrate
npm run db:seed
npm run build
```

업로드 폴더 권한 (최초 1회):

```bash
mkdir -p /var/www/Cliva-RC_promotion/booth-roulette/server/uploads/promo
chown -R $(whoami):$(whoami) /var/www/Cliva-RC_promotion/booth-roulette/server/uploads
```

DB 백업 (KOIHA와 동일 방식):

```bash
mkdir -p /root/backups
cd /var/www/Cliva-RC_promotion/booth-roulette/server
export $(grep -v '^#' .env | grep DATABASE_URL | xargs)
ts=$(date +%Y%m%d_%H%M%S)
pg_dump "$DATABASE_URL" -Fc -f "/root/backups/booth_roulette_${ts}.dump"
```

행사 후 **홍보 영상·DB** 함께 백업 권장:

```bash
tar -czf "/root/backups/booth_uploads_${ts}.tar.gz" \
  -C /var/www/Cliva-RC_promotion/booth-roulette/server uploads
```

## 7. Nginx (KOIHA nginx-cliva-edu.example.conf 참고)

부스 전용 서브도메인 예 (`promotion.cliva.net` → `127.0.0.1:4001`):

```nginx
upstream booth_api {
    server 127.0.0.1:4001;
    keepalive 16;
}

server {
    listen 443 ssl http2;
    server_name promotion.cliva.net;

    root /var/www/Cliva-RC_promotion/booth-roulette/client/dist;
    index index.html;

    # API
    location /api/ {
        proxy_http_version 1.1;
        proxy_set_header Host              $host;
        proxy_set_header X-Real-IP         $remote_addr;
        proxy_set_header X-Forwarded-For   $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_pass http://booth_api;
    }

    # 홍보 영상 (관리자 업로드 파일)
    location /uploads/ {
        proxy_http_version 1.1;
        proxy_set_header Host              $host;
        proxy_set_header X-Real-IP         $remote_addr;
        proxy_set_header X-Forwarded-For   $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_pass http://booth_api;
        client_max_body_size 500M;
    }

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

`client_max_body_size 500M` — 영상 업로드 용량 제한과 맞춥니다.

```bash
sudo nginx -t && sudo systemctl reload nginx
curl -sS https://promotion.cliva.net/api/health
```
